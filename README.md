# tr — 라이트노벨 DOCX 번역기

일본어 라이트노벨 DOCX 파일을 한국어로 번역하는 CLI 도구.  
Google Gemini API (무료) 또는 Vertex AI를 선택해서 사용할 수 있다.

## 특징

- **다중 작품 관리** — `books/` 폴더 아래 작품별로 독립 프로젝트 구성
- **용어집** — 고유명사·작품 용어를 CSV로 지정해 일관되게 번역
- **호칭 룰북** — 캐릭터 간 호칭을 CSV로 지정해 말투 일관성 보장
- **체크포인트** — 10 청크마다 자동 저장, 중단 후 이어서 번역 가능
- **일본어 잔존 재시도** — 번역 결과에 일본어가 남으면 최대 3회 자동 재시도
- **이미지 번역** — Vertex AI 백엔드 선택 시 사용 가능

## 요구 사항

- Python 3.11 이상
- Google API 키 (백엔드에 따라 아래 중 하나)

| 백엔드 | 환경변수 | 비고 |
|--------|---------|------|
| Gemini 무료 API | `GOOGLE_API_KEY` | 이미지 번역 미지원 |
| Vertex AI | `GOOGLE_CLOUD_API_KEY` | 이미지 번역 지원, 유료 |

환경변수가 없으면 실행 시 입력 프롬프트가 뜬다.

## 설치

```bash
pip install -r requirements.txt
```

## 실행

```bash
python main.py
```

실행하면 순서대로 선택 UI가 나온다.

1. **백엔드 선택** — Gemini 무료 API / Vertex AI
2. **API 키 확인** — 환경변수 없으면 입력 요청
3. **작품 선택** — `books/` 아래 폴더 목록
4. **파일 선택** — 선택한 작품의 `source/` 안 파일 목록
5. **추론 수준 선택** — 최소 / 낮음 / 보통 / 높음
6. **이미지 번역 여부** — Vertex AI 선택 시에만 표시
7. **광고 이미지 포함 여부** — 이미지 번역 켜기 선택 시에만 표시

번역 결과는 해당 작품의 `result/` 폴더에 저장된다.

## 작품 폴더 구조

```
books/
└── 작품명/
    ├── source/          ← 번역할 원본 DOCX 파일을 여기에 넣는다
    ├── result/          ← 번역 결과 및 체크포인트 파일이 저장된다
    ├── glossary.csv     ← 용어집
    └── honorifics.csv   ← 호칭 룰북
```

새 작품을 추가하려면 `books/작품명/` 폴더를 만들고 위 구조에 맞게 파일을 넣으면 된다.

## 용어집 (glossary.csv)

일본어 원문과 번역어를 쉼표로 구분해서 작성한다. 첫 줄은 헤더.

```csv
ja,ko
ベル,벨
ヘスティア,헤스티아
ファミリア,파밀리아
```

## 호칭 룰북 (honorifics.csv)

캐릭터 A가 캐릭터 B를 부를 때 사용할 호칭을 지정한다. 첫 줄은 헤더.

```csv
caller,target,honorific
벨,아이즈,아이즈 씨
벨,헤스티아,헤스티아 님
```

- `caller` — 부르는 쪽 캐릭터 이름
- `target` — 불리는 쪽 캐릭터 이름
- `honorific` — 사용할 호칭 (이 문자열 그대로 사용됨, 뒤에 추가 접미사 없음)

## 추론 수준

모델의 thinking 수준을 조절한다. 수준이 높을수록 번역 품질이 올라가지만 처리 시간이 길어진다.  
일반적으로 **최소**로도 충분하다.

| 선택 | 내부값 |
|------|--------|
| 최소 | MINIMAL |
| 낮음 | LOW |
| 보통 | MEDIUM |
| 높음 | HIGH |

## 체크포인트

번역 도중 중단되거나 오류가 발생하면 `result/[checkpoint] 파일명.json`에 진행 상황이 저장된다.  
다시 실행하면 이어서 번역할지 물어본다. 완료 시 체크포인트 파일은 자동으로 삭제된다.
