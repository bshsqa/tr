# DEV-001: Gemini 무료 API 전환 및 번역 품질 개선

## 배경

기존 코드는 Vertex AI API를 사용하고 있었으나, Google 무료 API(`gemini-3.1-flash-lite`)로 전환하고 전반적인 번역 품질을 개선한다.

---

## 변경 사항

### 1. API 전환

- Vertex AI → Google 무료 API
- 환경변수: `GOOGLE_CLOUD_API_KEY` → `GOOGLE_API_KEY`
- `genai.Client(vertexai=True)` → `genai.Client(api_key=...)`

### 2. 모델 변경

- 텍스트 모델: `gemini-3-flash-preview` → `gemini-3.1-flash-lite`
- 이미지 번역 모델(`gemini-3-pro-image-preview`)은 무료 API 미지원으로 호출하지 않음
- 이미지는 항상 원본 그대로 출력 문서에 유지됨

### 3. 프롬프트 전면 개선

다음 규칙을 프롬프트에 명시적으로 주입:

1. 히라가나·가타카나·한자를 포함한 일본어 문자를 출력에 남기지 않을 것
2. Glossary에 등록된 용어는 반드시 지정된 한국어 표현으로 번역할 것
3. 간투어(えっと→음..., ええ→에..., あの→저... 등)를 자연스러운 한국어로 대체
4. 의성어·의태어(擬音語·擬態語)를 적절한 한국어 표현으로 대체
5. 캐릭터별 말투를 문맥에서 파악하여 끝까지 일관되게 유지
6. 라이트노벨 전문 번역가 수준의 자연스러운 한국어 출력
7. 줄바꿈 유지, 「」 등 기호 원형 유지

### 4. honorifics.csv 룰북

캐릭터 간 호칭을 명시적으로 지정하는 룰북 파일.

**형식:**
```
caller,target,honorific
누쿠미즈,야나미,야나미 양
카쥬,야나미,야나미 언니
```

- `caller`: 부르는 캐릭터 (한국어 이름)
- `target`: 불리는 캐릭터 (한국어 이름)
- `honorific`: 번역에서 사용할 호칭 전체 (경칭 포함)

**주의:** honorific 값은 경칭이 이미 포함된 완성형이므로, LLM이 さん·상·씨 등을 추가로 붙이지 않도록 프롬프트에서 명시한다.

### 5. 일본어 잔존 시 재시도 메커니즘

번역 결과에 일본어 문자가 감지되면 최대 3회까지 재시도한다.

```
1차 번역
  ↓ 일본어 감지 (정규식: [぀-ヿ一-鿿])
  ↓ 발견 시 → 잔존 위치 주변 문맥 추출
2차 시도: 잔존 위치 힌트 포함하여 재요청
  ↓ 또 발견 시
3차 시도
  ↓ 모두 실패 시 → 일본어가 가장 적게 남은 결과 사용
```

### 6. 이미지 번역 UI 제거

이미지 번역 옵션(이미지 번역 여부, 광고 이미지 여부)을 UI에서 제거한다.
이미지는 항상 원본 그대로 출력에 포함된다.
